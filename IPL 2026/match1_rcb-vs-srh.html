RCB vs SRH – Match Analysis

Turning Points

  • Duffy’s first spell: Early damage, not a collapse. SRH lost three attacking options in the first five overs, shifting the innings from multiple threats to dependence on one player.
  • Kishan out at 80 (16th over): The defining moment. He was set to control the death overs. Once he fell, SRH had hitters but no one controlling the innings.
  • Kohli dropped at 28: The only real opening in the chase. Once missed, the game became straightforward.

Phase Control

Powerplay:
SRH: Wickets hurt them more than the runs helped. They were adjusting, not dictating.
RCB: Clean hitting, no pressure. Got ahead of the game early.
→ RCB clearly ahead.

Middle Overs:
SRH: Kishan carried the innings; others played around him.
RCB: No risk required. Kohli paced it with support.
→ RCB slightly ahead as the game never tightened.

Death Overs:
SRH: Strong finish, but catching up rather than pushing ahead.
RCB: No pressure phase reached.
→ RCB had already taken the game away.

Match Decider

RCB’s powerplay with the bat
That 70+ start removed scoreboard pressure and forced SRH out of attacking options early. From there, the chase followed a set path.

Decision Insight

SRH didn’t react quickly enough in the powerplay. It wasn’t just personnel, but predictability. Same lengths, limited field adjustments, and no attempt to break Padikkal’s rhythm allowed the phase to drift.

Win Probability Shift (directional)

  • SRH 30/3: RCB ahead
  • Kishan rebuild: SRH slightly ahead
  • Kishan out: RCB back in control
  • RCB powerplay: strong shift to RCB
  • Kohli drop: game almost done

Key point: SRH had moments, but never held control long enough.

Framework Insights

  • Early wickets need follow-through: Taking wickets is step one; controlling the next phase is what matters.
  • Powerplay in a chase defines the game: Getting ahead early removes the need for risk later.
  • Set batter at the death is the biggest asset: Kishan’s wicket mattered because of timing, not just value.
  • Pressure must be created: If the game drifts, you need to force change—field, pace, or angle. SRH didn’t.
